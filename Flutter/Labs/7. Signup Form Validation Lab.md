# Lab 7: Building a Signup Form with Validation & Good UX

## Objectives

Thiết kế và implement một user registration (signup) form thu thập input, validate dữ liệu và cung cấp phản hồi rõ ràng cho người dùng.

> [!info] Module liên quan
> Lab này củng cố các kiến thức từ **[[7. Forms & Validation]]**, bao gồm:
> - Dùng `Form` và `TextFormField` cho structured input
> - Dùng `FormState` và `GlobalKey` để validate và save data
> - Áp dụng built-in và custom validation rules
> - Quản lý focus và keyboard behaviors trên mobile
> - Cải thiện form UX với error messages và button states
> - (Optional) Async validation trước khi tạo account

## Requirements Checklist

**Lab 7.1 — Basic Registration Form:**
- [ ] Field: Full Name
- [ ] Field: Email
- [ ] Field: Password
- [ ] Field: Confirm Password
- [ ] Submit button

**Lab 7.2 — Validation Rules & Password Strength:**
- [ ] Tất cả fields là required
- [ ] Email phải có định dạng hợp lệ (có @ và .)
- [ ] Password: tối thiểu 8 ký tự, ít nhất 1 chữ số
- [ ] Confirm password phải khớp với password
- [ ] Hiển thị inline error messages

**Lab 7.3 — Focus & Keyboard Management:**
- [ ] `FocusNode` cho từng field
- [ ] `TextInputAction.next` / `.done` phù hợp
- [ ] Tab qua các fields: Name → Email → Password → Confirm
- [ ] Tap outside form để dismiss keyboard

**Lab 7.4 — Optional Async Validation:**
- [ ] Giả lập API check email (dùng `Future.delayed`)
- [ ] Email bắt đầu bằng "taken" → hiển thị lỗi
- [ ] Disable submit button và hiển thị loading khi đang check

## Guided Steps

### Step 1 — Project Setup

```bash
flutter create lab7_forms_validation
```
Hoặc mở **DartPad → New Flutter**. Verify app chạy với "Hello" text trước khi thêm logic.

---

### Step 2 — Create the Basic Form Layout (Lab 7.1)

```dart
final formKey = GlobalKey<FormState>();

// Structure:
// Scaffold → AppBar → body → Padding → Form → ListView
//   → TextFormField (Full Name)
//   → TextFormField (Email)
//   → TextFormField (Password)
//   → TextFormField (Confirm Password)
//   → ElevatedButton (Submit)
```

> [!note] Lúc này form chưa có validation — chỉ cần thu thập input.

---

### Step 3 — Add Basic Validation (Lab 7.1 → 7.2)

```dart
// Trong onPressed của Submit button:
final isValid = formKey.currentState!.validate();
if (isValid) {
  formKey.currentState!.save();
  // Show SnackBar
}
```

**Validators cơ bản:**
- Required field: `if (value == null || value.isEmpty) return 'Trường này là bắt buộc';`
- Email: kiểm tra có `@` và `.`

---

### Step 4 — Implement Password Rules (Lab 7.2)

```dart
String? validatePassword(String? value) {
  if (value == null || value.length < 8) return 'Tối thiểu 8 ký tự';
  if (!RegExp(r'[0-9]').hasMatch(value)) return 'Phải có ít nhất 1 chữ số';
  return null;
}
```

Bật `autovalidateMode: AutovalidateMode.onUserInteraction` trên Form để feedback nhanh hơn.

---

### Step 5 — Manage Focus & Keyboard (Lab 7.3)

```dart
// Tạo FocusNode cho mỗi field
final nameFocus = FocusNode();
final emailFocus = FocusNode();
final passwordFocus = FocusNode();
final confirmFocus = FocusNode();

// Trong TextFormField:
textInputAction: TextInputAction.next,
onFieldSubmitted: (_) => FocusScope.of(context).requestFocus(emailFocus),

// Wrap toàn bộ screen trong GestureDetector:
onTap: () => FocusScope.of(context).unfocus(),
```

> [!warning] Tránh overflow
> Wrap form trong `ListView` hoặc `SingleChildScrollView` để tránh overflow khi bàn phím mở.

---

### Step 6 — Async Email Check (Optional, Lab 7.4)

```dart
bool isCheckingEmail = false;

Future<void> _submit() async {
  if (!formKey.currentState!.validate()) return;
  
  setState(() => isCheckingEmail = true);
  
  await Future.delayed(const Duration(seconds: 2));
  final email = emailController.text;
  
  if (email.startsWith('taken')) {
    ScaffoldMessenger.of(context).showSnackBar(
      const SnackBar(content: Text('Email này đã được sử dụng')),
    );
  }
  
  setState(() => isCheckingEmail = false);
}
```
