> [!note]+ # Tổng quan
> > [!note]+ ### JavaScript là gì?
> > > JS là ngôn ngữ lập trình web, nó có thể thay đổi lên HTML và CSS, thêm vào đó là tính toán, thao túng và chuẩn hoá dữ liệu.
> > 
> > Ba ngôn ngữ lập trình web bắt buộc phải học: HTML, CSS, JS
> > [!note]+ ### Cách để thêm JS
> > - **Cách 1: JS Nội bộ (Internal)**
> > JS có thể được nhúng vào HTML ở trong `body` và `head` bằng thẻ `script`
> > ```html
> > <html>
> > 	<head>
> > 		<script>
> > 			//JavaScript code here
> > 		</script>
> > 	</head>
> > 	<body>
> > 		<script>
> > 			//JavaScript code here
> > 		</script>
> > 	</body>
> > </html>
> > ```
> > - **Cách 2: JS Bên Ngoài (External)**
> > Ta liên kết tới một file js từ bên ngoài bằng cú pháp:
> > ```html
> > <head>
> > 	<script src="js/script.js"></script>
> > </head>
> > ```

> > - Mở Trình duyệt bất kỳ
> > - Vào một trang bất kỳ nào đó
> > - Nhấn `F12` hoặc ‘chuột phải → Inspect`
> > - Ở DevTool, mở tab `Console`

> [!note]+ # Biến và kiểu dữ liệu
> > [!note]+ ### Biến
> > - **Các biến trong JS là các ‘khoang chứa dữ liệu’. Có 3 cách để khai báo biến:**
> >     - `var` (cách truyền thống, không còn được khuyến khích sử dụng)
> > ```javascript
> > var message = "Hello";
> > var count = 10;
> > var isActive = true;
> > ```
> >     - `let` (ES6, sử dụng cho các biến có thể thay đổi)
> > ```javascript
> > let message = "Hello";
> > let count = 10;
> > let isActive = true;
> > ```
> >     - `const` (ES6, dành cho các hằng biến không đổi)
> > ```javascript
> > const PI = 3.14;
> > const PATTERN = /^[A-Za-z0-9_]{5, 20}$/;
> > ```
> > - **Quy tắc đặt tên biến:**
> >     - Phải bắt đầu với chữ cái `letter` , gạch dưới `_` hoặc ký tự `$`
> >     - Có thể bao gồm chữ cái, số, gạch dưới và ký tự `$`
> >     - **Case-sensitive** (Phân biệt hoa - thường)
> >     - Không thể đặt trùng với keyword có sẵn của JS
> > ```javascript
> > // Valid variable names
> > let firstName = "Minh";
> > let _count = 5;
> > let $price = 10.99;
> > let camelCase = "This is a common naming convention";
> > 
> > // Invalid variable names
> > // let 123abc = "Cannot start with a number";
> > // let my-name = "Hyphens are not allowed";
> > // let let = "Cannot use reserved keywords";
> > ```
> >

> > [!note]+ ### Kiểu chuỗi (String)
> > - Kiểu chuỗi dùng để lưu trữ văn bản (dùng `""` hoặc `''` đều được)
> > ```javascript
> > let greeting = "Hello";
> > let courseName = 'JavaScript';
> > let backtick = `Template literal`;
> > // String concatenation
> > let message = greeting + " " + courseName;
> > // Template literals
> > let template = `${greeting} ${courseName}!`;
> > // Useful methods
> > console.log(courseName.length); // 10
> > console.log(courseName.toUpperCase()); // "JAVASCRIPT"
> > console.log(courseName.substring(0, 4)); // "Java"
> > console.log(courseName.indexOf("Script")); // 4
> > ```
> > - Một số phương thức cho String
> >     - `str.trim()`: Loại bỏ khoảng trắng ở đầu và cuối chuỗi
> >     - `str.toLowerCase()`: Chuyển toàn bộ chuỗi thành chữ thường
> >     - `str.toUpperCase()`: Chuyển toàn bộ chuỗi thành chữ hoa
> >     - `str.includes(substr)`: Kiểm tra xem một chuỗi có bao gồm chuỗi khác không
> >     - `str.split(regex)`: Tách chuỗi thành một mảng các chuỗi, cách nhau bởi kí tự
> >     - `str.substring(start, end)`: Lấy ra chuỗi con từ index start đến end
> > [!note]+ ### Kiểu logic (Boolean)
> > - Kiểu Logic chỉ có hai giá trị: `True` hoặc `False`
> > ```javascript
> > let isActive = true;
> > let isCompleted = false;
> > // Comparison operators
> > console.log(5 > 3);
> > console.log(5 < 3);
> > console.log(5 >= 5);
> > console.log(5 === "5"); // strict equality, checks both value and type loose
> > console.log(5 == "5"); //  equality, compares values but allows type coercion
> > console.log(5 !== 10);
> > ```
> > [!note]+ ### Kiểu mảng (Array)
> > - Kiểu mảng là một biến đặc biệt, nắm giữ nhiều hơn 1 giá trị
> > ```javascript
> > let colors = ["red", "green", "blue"];
> > let mixed = [1, "text", true, null, {name: "object"}, [1, 2]];
> > // both create a new empty array named points:
> > const points = new Array();
> > const points = [];
> > // both create a new array containing 6 numbers:
> > const points = new Array(40, 100, 1, 5, 25, 10);
> > const points = [40, 100, 1, 5, 25, 10];
> > // Access elements
> > console.log(colors[0]);
> > console.log(colors.length);
> > // A Common Error
> > const points = [60]; // OK
> > // is not the same as:
> > const points = new Array(60); // NOK: Avoid using new Array().
> > ```
> > - Một số phương thức cho mảng:
> >     - `arr.push(x)`: Đẩy một hoặc nhiều giá trị vào cuối mảng
> >     - `arr.pop()`: Lấy giá trị cuối cùng ra khỏi mảng (trả về giá trị đó)
> >     - `arr.shift()`: Lấy ra giá trị đầu tiên trong mảng (trả về giá trị đó)
> >     - `arr.unshift(x)`: Thêm một hoặc nhiều giá trị vào đầu mảng
> >     - `arr.map(lambda)`: Biến đổi thành mảng mới dựa trên quy tắc nào đó
> > ```javascript
> > [1, 2, 3].map(x => x * 2) //output: [2, 4, 6]
> > ```
> >     - `arr.filter(lambda)`: Lọc ra một mảng con với các giá trị thoả mãn quy tắc nào đó
> > ```javascript
> > [1, 2, 3].filter(x => x > 1) //output: [2, 3]
> > ```
> >     - `arr.find(lambda)`: Trả về phần tử đầu tiên phù hợp với bắt đầu
> > ```javascript
> > users.find(u => u.id === 1) //output: { id: 1, name: "A" }
> > ```
> >     - `arr.forEach(lambda)`: Thực thi một hàm vô danh cho mỗi phần tử
> > ```javascript
> > [1, 2, 3].forEach(x => console.log(x)) //In ra từng phần tử
> > ```
> > [!note]+ ### Kiểu đối tượng (Object)
> > - Trong JS, một đối tượng là một kiểu dữ liệu lưu trữ dưới dạng cặp ‘key - value’
> > - Một đối tượng được khởi tạo bằng dấu `{}` và mỗi thuộc tính là một cặp K-V
> > - Có thể được khởi tạo bằng từ khoá `new` và xoá bằng từ khoá `delete`
> > 
> > ```javascript
> > let person = {
> > 	firstName: "Minh",
> > 	lastName: "Nguyen",
> > 	age: 22,
> > 	isStudent: true,
> > 	address: {
> > 		city: "Hanoi",
> > 		country: "Vietnam"
> > 	},
> > 	hobbies: ["reading", "coding"]
> > };
> > 
> > // Access properties
> > console.log(person.firstName);
> > console.log(person["lastName"]);
> > console.log(person.address.city);
> > console.log(person.hobbies[1]);
> > 
> > // Add new property
> > person.email = "minh@example.com";
> > 
> > // Delete property
> > delete person.isStudent;
> > ```
> > - Phương thức của đối tượng:
> >     - Phương thức của đối tượng là hành động mà đối tượng có thể thực hiện
> >     - Phương thức được định nghĩa dưới dạng K-V như một thuộc tính
> > ```javascript
> > const person = {
> > 	firstName: "John",
> > 	lastName: "Doe",
> > 	id: 5566,
> > 	fullName: function() {
> > 		return this.firstName + " " + this.lastName;
> > 		}
> > 	};
> > 	
> > // Accessing Object Methods
> > name = person.fullName();
> > 
> > // Create an Array
> > const myArray = Object.values(person);
> > ```
> > - Một số phương thức có sẵn:
> >     - `Object.keys(obj)`: trả về một mảng chỉ chứa các key
> >     - `Object.values(obj)`: trả về một mảng chỉ chứa các value
> >     - `Object.entries(obj)`: trả về một mảng chứa các cặp K-V
> >     - `obj.hasOwnProperty(key)`: kiểm tra xem đối tượng có một key hay không
> >     - `JSON.stringify()`: Chuỗi hoá một đối tượng
> > ```javascript
> > let myString = JSON.stringify(person);
> > ```

> [!note]+ # Điều kiện và Lặp
> > [!note]+ ### Điều kiện
> > > Hầu hết cú pháp điều kiện đều giống Java
> > 
> > - `If…else`
> > ```javascript
> > let score = 85;
> > if (score >= 90) {
> > 	console.log("Excellent");
> > } else if (score >= 70) {
> > 	console.log("Good");
> > } else {
> > 	console.log("Try harder next time");
> > }
> > ```
> > - `Switch…case`
> > ```javascript
> > let color = "green";
> > switch (color) {
> > 	case "red":
> > 		console.log("Stop!");
> > 		break;
> > 	case "green":
> > 		console.log("Go!");
> > 		break;
> > 	case "yellow":
> > 		console.log("Slow down!");
> > 		break;
> > 	default:
> > 		console.log("Invalid color");
> > }
> > ```
> > - Ternary Operator
> > ```javascript
> > //SYNTAX: (condition) ? (run if true) : (run if false)
> > let age = 18;
> > let access = age >= 18 ? "Allowed" : "Denied";
> > console.log(access); // "Allowed"
> > ```
> > [!note]+ ### Lặp
> > - `for`
> > ```javascript
> > for (let i = 1; i <= 10; i++) {
> > 	if (i === 5) {
> > 	break; //exit the loop completely
> > 	}
> > 	console.log(i); // Prints 1 to 4
> > }
> > ```
> > - `while`
> > ```javascript
> > let i = 0;
> > 
> > while (i < cats.length){
> > 	if (i === cats.length - 1) {
> > 		info += 'and ' + cats[i] + '.';
> > 	} else {
> > 		info += cats[i] + ',';
> > 	}
> > 	i++
> > }	
> > ```
> > - `do...while`
> >     - Tương tự như lặp while, nhưng chắn chắn sẽ chạy qua một vòng lặp trước khi kiểm tra điều kiện
> > - `for...of` (giống với `for :` )
> >     - Lặp qua các giá trị của kiểu dữ liệu (Array, List,…)
> > ```javascript
> > for (let cat of cats) {
> > 	console.log(cat);
> > }
> > ```
> > - `for...in` (giống với việc duyệt qua `keySet` của `Map`)
> >     - Lặp qua các key hoặc value của một Object
> > ```javascript
> > let person = {
> > 	name: "Minh",
> > 	age: 22,
> > 	city: "Hanoi"
> > };
> > 
> > for (let key in person) {
> > 	console.log(`${key}: ${person[key]}`);
> > }
> > ```
> [!note]+ # Hàm
> > [!note]+ ### Khai báo hàm
> > - Hàm trong JS là một khối code được tạo ra để thực thi một công việc nhất định
> > - Một hàm trong JS chỉ được thực thi khi có “cái gì đó” GỌI đến nó
> > - Việc khai báo hàm có syntax như sau:
> > ```javascript
> > function function_name(parameter){
> > 	return result;
> > }
> > ```
> > 
> > > [!note] 🗣
> > > 
> > > 1. Khai báo một hàm không thực thi hàm đó
> > > 2. Khi gọi hàm, ta phải truyền tham số đúng với hàm đã tạo
> > > 3. Khi ta sử dụng hàm để tương tác lên đối tượng bên ngoài, đối tượng đó sẽ bị thay đổi vĩnh viễn
> > [!note]+ ### Biểu thức hàm
> > - Biểu thức hàm được dùng để tạo ra một hàm ẩn danh (dùng 1 lần và không có tên)
> > ```javascript
> > const square = function(number) { return number * number }
> > var x = square(4)
> > ```
> > - Ta cũng có thể cung cấp tên cho hàm ẩn danh, chủ yếu là để hàm đó gọi lại chính nó
> > ```javascript
> > const factorial = function fac(n) { return n < 2 ? 1 : n * fac(n-1) }
> > ```
> > - Biểu thức hàm là việc coi hàm như một biểu thức, giúp ta truyền vào một hàm khác.
> > ```javascript
> > function map(f,a) {
> > 	let result = [];
> > 	let i;
> > 	for (i = 0; i != a.length; i++){
> > 		result[i] = f(a[i]);
> > 	}
> > 	return result;
> > }
> > 
> > const f = function(x) {
> > 	return x * x * x;
> > }
> > 
> > let numbers = [0,1,2,3,4,5];
> > let cube = map(f,numbers);
> > console.log(cube);
> > ```
> > - Ta cũng có thể dùng điều kiện để kiểm soát việc gán biến cho hàm.
> > ```javascript
> > var myFunc;
> > if (num === 0){
> > 	myFunc = function(obj) {
> > 		obj.make = 'Toyota';
> > 	}
> > }
> > ```
> > [!note]+ ### Call Stack
> > > Call Stack là cơ chế của trình thông dịch để kiểm soát luồng chạy khi gọi nhiều hàm. Tức là nó theo dõi hàm nào đang được chạy, hàm nào sẽ được gọi tiếp.
> > 
> > - Tuân theo cơ chế LIFO của Stack

> > [!note]+ ### Hàm Đệ Quy
> > > Sự đệ quy là khi hàm gọi lại chính nó, mục đích để xử lý những vấn đề mà trong đó bao gồm các vấn đề nhỏ khác. Đệ quy có 2 trường hợp xảy ra:
> > 1. Trường hợp cơ bản (Kết thúc đệ quy) hay `điều kiện dừng`
> > 2. Trường hợp đệ quy (Tiếp tục đệ quy)

> [!note]+ # DOM
> > [!note]+ ### Tổng quan
> > > **DOM (Document Object Model)** là **một mô hình đối tượng** dùng để biểu diễn cấu trúc của tài liệu HTML hoặc XML dưới dạng **cây (tree)**.
> > 
> > - Mỗi thành phần của trang web (thẻ HTML, thuộc tính, văn bản, comment…) được biểu diễn dưới dạng **Node**. Node có thể là **Element Node**, **Text Node**, **Attribute Node**, v.v.
> > - DOM cho phép **các ngôn ngữ lập trình** (thường là JavaScript) **truy cập, thay đổi, xóa hoặc thêm các phần tử** trên trang một cách linh hoạt.
> > 
> > ```javascript
> > const paragraphs = document.getElementByTagName("p");
> > // hàm này trả về HTMLCollection, có thể được truy cập thông qua index
> > //VD: paragraphs[0] là phần tử đầu tiên
> > //VD: paragraphs[1] là phần tử thứ hai
> > ```
> > [!note]+ ### Cấu trúc
> > > DOM biểu diễn tài liệu HTML dưới dạng cây phân cấp:
> > 
> > - Đối tượng `document` là đối tượng ở **Root Node**.
> > - Các thẻ trở thành **Element Node**.
> > - Văn bản bên trong các thẻ trở thành **Text Node**.
> > - Các thuộc tính (Style, class,…) trở thành **Attribute Node**.
> > - Các comment trở thành **Comment Node**
> > 
> > ![[image 36.png]]
> > Vì mọi phần tử bên trong DOM đều trở thành Node, ta truy cập chúng qua `.`
> > ```javascript
> > let nodes = document.getElementByTagName("p").childNodes;
> > ```
> > [!note]+ ### Truy cập phần tử DOM
> > > [!note]+ Cơ bản
> > > ```javascript
> > > // Tìm bằng ID (để trả về phần tử duy nhất)
> > > const element = document.getElementById('myId');
> > > // Tìm bằng Class (để trả về Live HTMLCollection các phần tử)
> > > const elements = document.getElementsByClassName('myClass');
> > > // Tìm bằng tên thẻ (để trả về Live HTMLCollection các phần tử)
> > > const paragraphs = document.getElementsByTagName('p');
> > > // Dùng cú pháp CSS Selector (để trả về phần tử đầu tiên tìm thấy)
> > > const header = document.querySelector('#header');
> > > // Dùng cú pháp CSS Selector (để trả về toàn bộ Node)
> > > const buttons = document.querySelectorAll('.button');
> > > ```
> > > > [!note] 🗣
> > > > Cú pháp CSS Selector yêu cầu truyền vào selector của CSS, tức là:
> > > > 1. Đối với `id` ta phải dùng `#`id
> > > > 2. Đối với `class` ta phải dùng `.class`
> > > > 3. Đối với các thẻ chồng nhau, ta dùng ví dụ: `div p`
> > 
> > > [!note]+ Truy cập phần tử Cha
> > > ```javascript
> > > const parent = element.parentNode;
> > > const parent2 = element.parentElement; // Only returns Element nodes
> > > ```
> > > - `parentNode` trả về Node cha của Node hiện tại, có thể là bất cứ loại nào (Element, Text, Comment, Text,…)
> > > - `parentElement` chỉ trả về Node cha của Node hiện tại nếu như nó là `Element Node`. Nếu như nó không phải là Element, trả về `null`.
> > 
> > > [!note]+ Truy cập phần tử Con
> > > ```javascript
> > > const children = element.childNodes; // All child nodes
> > > const childElements = element.children; // Only Element nodes
> > > const firstChild = element.firstChild; // First child node (any type)
> > > const firstElement = element.firstElementChild; // First child Element
> > > ```
> > > - `childNodes` trả về NodeList bao gồm tất cả các Node con của Node đó.
> > > - `children` trả về HTMLCollection bao gồm các Node con `Element`.
> > > - `firstChild` trả về Node con đầu tiên (bất kể loại của nó)
> > > - `firstElementChild` trả về Node con `Element` đầu tiên 
> > 
> > > [!note]+ Truy cập phần tử Anh Em
> > > ```javascript
> > > const nextSibling = element.nextSibling; // Next sibling node (any type)
> > > const nextElement = element.nextElementSibling; // Next sibling Element
> > > const prevSibling = element.previousSibling; // Previous sibling node
> > > const prevElement = element.previousElementSibling; // Prev sibling Element
> > > ```
> > > - `nextSibling` trả về node tiếp theo cùng cấp
> > > - `nextElementSibling` trả về node Element tiếp theo cùng cấp, bỏ qua các loại khác
> > > - `prevSibling` trả về node trước đó cùng cấp
> > > - `prevElementSibling` trả về node Element trước đó cùng cấp, bỏ qua các loại khác
> > 
> > > [!note]+ Một số điểm cần lưu ý
> > > - `Node` và `Element`: Trong DOM, mọi phần tử đều là `Node` , nhưng chưa chắc mọi Node đã là một `Element` . Element là các Node thường thấy dưới dạng thẻ HTML như `<div>`, `<p>`,…
> > > - Các khoảng trắng và cách xuống dòng trong tài liệu HTML mà nằm giữa 2 thẻ HTML cũng được tính là `Text Node`. Khi dùng các phương thức truy cập phần tử DOM như trên, ta dễ dàng bắt gặp chúng nếu không yêu cầu cụ thể `Element`
> > > - Collection và Array: phương thức `childNodes` và `children` trả về `Collection` (NodeList và HTML Collection), không trả về `Array`. Chúng giống Array nhưng không có phương thức để duyệt qua như `forEach()` hay biến đổi như `map()` 
> > > (NodeList đời mới có forEach nhưng HTMLCollection thì không)
> > 
> > 
> > [!note]+ ### Thao túng phần tử DOM
> > > [!note]+ Cách điệu phần tử
> > > - Styling cùng dòng:
> > > ```javascript
> > > // Inline styling
> > > element.style.color = 'blue';
> > > element.style.backgroundColor = '#f0f0f0';
> > > element.style.fontSize = '16px';
> > > ```
> > > - Thao túng Class (Áp dụng cho Bootstrap 5)
> > > ```javascript
> > > // Class manipulation
> > > element.classList.add('active');
> > > element.classList.remove('hidden');
> > > element.classList.toggle('visible');
> > > element.classList.contains('selected'); // Returns boolean
> > > ```
> > > Trong đó:
> > >     - `element.classList.add('...')`: sẽ thêm … vào thuộc tính class của phần tử.
> > >     - `element.classList.remove('...')`: sẽ xóa … khỏi thuộc tính class của phần tử.
> > >     - `element.classList.toggle('...')`: giống như việc bật/tắt, phương thức này chuyển giữa trạng thái class `visible` hoặc không có `visible`
> > >     - `element.classList.contains('...')`: Kiểm tra xem phần tử có thuộc tính class ‘…’ hay không.
> > 
> > > [!note]+ Tạo ra phần tử
> > > ```javascript
> > > // Create a new element
> > > const newDiv = document.createElement('div');
> > > // Create a text node
> > > const newText = document.createTextNode('Hello, World!');
> > > // Combine them
> > > newDiv.appendChild(newText);
> > > // Add to the document
> > > document.body.appendChild(ncode
> > > ```
> > > - `document.createElement('...')`: sẽ tạo ra một phần tử mới với thẻ <…>. Phần tử này sẽ được tạo ra và lưu trong bộ nhớ, nhưng chưa gắn vào DOM.
> > > - `createTextNode('...')`: sẽ tạo ra một `Text Node` chứa chuỗi truyền vào. Tương tự như trên, tồn tại trong bộ nhớ nhưng chưa gắn vào DOM.
> > > - `document.body.appendChild(newDiv)`: Phương thức này gắn Node `newDiv` vào danh sách Node con của Node cha (ở đây là body). Tức là tạo ra một bộ thẻ newDiv mới:
> > > ```javascript
> > > <newDiv></newDiv>
> > > ```
> > > - `newDiv.appendChild(newText)`: phương thức này gắn `Text Node` vừa được tạo ra vào bên danh sách Node con của Node cha `newDiv`. Nói dễ hiểu, bây giờ trong thẻ `newDiv` của ta sẽ có chuỗi `newText`: 
> > > ```javascript
> > > <newDiv> 123text123 </newDiv>
> > > ```
> > 
> > > [!note]+ Chỉnh sửa phần tử
> > > - Sửa nội dung
> > >     - `element.textContent = 'New text content'`: sẽ gán Text Node con của một phần tử bằng chuỗi mới. Nếu chưa có Text Node, nó sẽ tạo mới.
> > >     - `element.innerHTML = '<span>HTML content</span>'`: sẽ gán nội dung bên trong của một phần tử bằng HTML được ép kiểu từ chuỗi ta truyền vào.
> > > **Note**: Sử dụng `innerHTML` với dữ liệu không đáng tin cậy sẽ dễ bị tấn công XSS
> > > ```javascript
> > > // Text only
> > > element.textContent = 'New text content'; 
> > > // Can include HTML (use carefully)
> > > element.innerHTML = '<span>HTML content</span>'; 
> > > ```
> > > - Làm việc với thuộc tính (Attributes)
> > >     - `element.setAttribute('...', '...')`: Sẽ gán thuộc tính bất kỳ của phần tử thành thuộc tính mới. Nếu chưa tồn tại, nó sẽ được tạo mới.
> > >     - `const attrValue = element.getAttribute('...')`: lấy ra thuộc tính nào đó của phần tử rồi lưu lại vào biến.
> > >     - `element.removeAttribute('...')`: Xóa thuộc tính nào đó của phần tử.
> > > ```javascript
> > > element.setAttribute('class', 'new-class');
> > > const attrValue = element.getAttribute('id');
> > > element.removeAttribute('disabled');
> > > ```
> > > Tương tự với Java, ta có thể sửa trực tiếp thuộc tính của phần tử mà không thông qua phương thức set/get:
> > > ```javascript
> > > element.id = 'newId';
> > > element.className = 'new-class'; // For the class attribute
> > > element.href = 'https://example.com'; // For <a> elements
> > > ```
> > 
> > > [!note]+ Loại bỏ phần tử
> > > Có 2 cách để xóa một phần tử khỏi DOM:
> > > - Xóa trực tiếp thông qua `element.remove()`
> > > - Xóa Node Con bằng `parent.removeChild(element)`
> > [!note]+ ### Các loại Node
> > - Có các loại Node sau đây:
> >     1. Element Node: là các thẻ HTML (e.g., <div>, <p>)
> >     2. Text Node: là phần văn bản nằm trong thẻ HTML
> >     3. Comment Node: là phần comment trong văn bản HTML
> >     4. Document Node: là Node lớn nhất bao gồm toàn bộ Node khác, đại diện cho cả văn bản HTML
> >     5. Các loại Node khác…
> > - Thuộc tính `nodeType`: là thuộc tính đại diện cho kiểu của Node
> >     1. nodeType === 1: Node đó là Element
> >     2. nodeType === 3: Node đó là Text
> >     3. nodenodeType === 8: Node đó là Comment
> >     4. nodeType === 9: Node đó là Document
> > **Note**: Các giá trị 1, 3, 8, 9,… ở trên là các hằng số đã được định nghĩa sẵn trong DOM
> > - Thuộc tính `nodeName` trả về tên của node đó.
> > 
> > > [!note]+ Ví dụ về các loại Node:
> > > > [!note]+ Element: là Node được dùng nhiều nhất
> > > > ```javascript
> > > > const div = document.getElementById("myDiv");
> > > > div.id = 'myDiv';
> > > > div.className = 'container';
> > > > div.setAttribute('data-custom', 'value');
> > > > ```
> > > > Một số phương thức hữu ích:
> > > > - `tagName`: Tag name (uppercase)
> > > > - `id, className, classList`: Identification and class properties
> > > > - `getAttribute(), setAttribute(), removeAttribute()`: Manage attributes
> > > > - `getElementsByTagName(), getElementsByClassName()`: Find child elements
> > > > - `innerHTML, outerHTML`: HTML content
> > > > - `textContent, innerText`: Text content
> > > 
> > > > [!note]+ Text
> > > > ```javascript
> > > > // Get the div element
> > > > const container = document.getElementById('container');
> > > > // Get the first text node in the div
> > > > const textNode = container.firstChild;
> > > > console.log(textNode.nodeType); // 3 (Text node)
> > > > console.log(textNode.nodeName); // "#text"
> > > > console.log(textNode.nodeValue); // "Hello World!" (along with whitespace)
> > > > // Create a new text node
> > > > const newTextNode = document.createTextNode('This is a new text node');
> > > > console.log(newTextNode.nodeType); // 3
> > > > console.log(newTextNode.nodeValue); // "This is a new text node"
> > > > // Append the text node to the end of the div
> > > > container.appendChild(newTextNode);
> > > > ```
> > > 
> > > > [!note]+ Comment
> > > > ```javascript
> > > > // Find a comment node
> > > > // One way is to iterate through body's childNodes
> > > > const bodyChildren = document.body.childNodes;
> > > > let commentNode;
> > > > for (let i = 0; i < bodyChildren.length; i++) {
> > > > 	if (bodyChildren[i].nodeType === 8) { // 8 = Comment node
> > > > 		commentNode = bodyChildren[i];
> > > > 		break;
> > > > 	}
> > > > }
> > > > // If comment is found
> > > > if (commentNode) {
> > > > 	console.log(commentNode.nodeType); // 8
> > > > 	console.log(commentNode.nodeName); // "#comment"
> > > > 	console.log(commentNode.nodeValue); // "This is a comment"
> > > > }
> > > > // Create a new comment node
> > > > const newComment = document.createComment('This is a new
> > > > comment');
> > > > console.log(newComment.nodeType); // 8
> > > > console.log(newComment.nodeValue); // "This is a new comment"
> > > > // Insert comment before the div
> > > > container.parentNode.insertBefore(newComment, container);
> > > > ```
> > >

> > [!note]+ ### Các kiểu Collection
> > 1. NodeList
> > > NodeList là tập hợp các Node, được trả về từ các phương thức như `querySelectorAll()` hay `childNodes`.
> > 
> > NodeList của một phần tử DOM là toàn bộ các Element, Text và Comment bên trong phần tử DOM đó.
> > ```javascript
> > const paragraphs = document.querySelectorAll('p'); // Static NodeList
> > const childNodes = element.childNodes; // Live NodeList
> > // Iterate through NodeList
> > paragraphs.forEach(p => {
> > p.classList.add('highlight');
> > });
> > ```
> > `NodeList động` và `NodeList tĩnh`:
> >     - Động: tự động cập nhật khi DOM thay đổi (`childNodes()`)
> >     - Tĩnh: chỉ trả về các Node tại thời điểm thực thi (`querySelectorAll()`)
> > Có thể chuyển về `Array`:
> > ```javascript
> > const paragraphArray = Array.from(paragraphs);
> > // Or
> > const paragraphArray2 = [...paragraphs];
> > ```
> > 2. HTMLCollection
> > > HTMLCollection là tập hợp các phần tử HTML, được trả ra từ các phương thức như: `getElementsByTagName()` , `getElementByClassName()` , `document.forms` , `form.elements` ,…
> > 
> > ```javascript
> > const divs = document.getElementsByTagName('div'); // Live HTMLCollection
> > // Iterate through HTMLCollection
> > for (let i = 0; i < divs.length; i++) {
> > 	divs[i].style.color = 'blue';
> > }
> > // Access via name (if id or name attribute exists)
> > const formElements = document.forms[0].elements;
> > const usernameInput = formElements.username; // Equivalent to
> > formElements['username']
> > ```
> > Khác với NodeList, HTMLCollection luôn động, tự động cập nhật khi DOM thay đổi.
> > Một số đặc điểm của HTML Collection:
> >     - **Không phải Array**: nó có method `.length` và truy xuất qua index, nhưng lại không có các phương thức như `.map()`, `.forEach()`, `.filter()`,…
> >     -  **Luôn Động**: nếu như DOM thay đổi (thêm hoặc xóa phần tử) thì danh sách phần tử sẽ được cập nhật ngay lập tức.
> >     - **Có thể truy cập phần tử qua tên hoặc id**
> >     - **Có thể chuyển về **`**Array**`
> > [!note]+ ### Tổng kết
> > | Interface | Purpose | Các phương thức ví dụ |
> > | --- | --- | --- |
> > | Document | Đại diện cho toàn thể văn bản HTML | getElementById(), querySelector() |
> > | Element | Đại diện cho các phần tử HTML | classList, setAttribute() |
> > | Node | Đại diện cho các Node (thẻ, văn bản,…) | appendChild(), removeChild() |
> > | Event | Đại diện cho các sự kiện trên trình duyệt | preventDefault(), stopPropagation() |
> [!note]+ # Event & Event Listening
> > [!note]+ ### Sự kiện là gì?
> > > Sự kiện là một hành động được xảy ra trên trang web, thường được kích hoạt bởi người dùng khi đang sử dụng trang web đó.
> > 
> > Các loại sự kiện thường gặp
> > 
> > | Event Name | When does it happen? | Example |
> > | --- | --- | --- |
> > | click | User clicks an element | Clicking a "Submit" button |
> > | mouseover | Mouse moves over an element | Hovering over an image |
> > | mouseout | Mouse moves out of an element | Moving cursor away from a button |
> > | keydown | User presses a key | Typing in an input field |
> > | submit | User submits a form | Submitting a registration form |
> > | change | Value of input/select changes | Selecting an option from dropdown |
> > | load | Page or image fully loads | Page finishes loading |
> > [!note]+ ### Các giai đoạn khi theo dõi Event
> > 1. Trình duyệt sẽ tạo một `đường đi` dẫn đến phần tử xảy ra sự kiện
> >     - VD: Event Path = [window, document, html, body, div, button]
> > 2. Capture: trình duyệt chạy xuống
> >     - Chạy xuống theo thứ tự các Node lớn → nhỏ 
> >     - Ở mỗi node, kiểm tra xem có Listener hay không
> >     - Nếu Listener có thuộc tính `capture: true` thì Listener sẽ chạy ở phase này
> > 3. Target
> >     - Trình duyệt đang ở đúng Node xảy ra sự kiện
> >     - Chạy cả các Listener có `capture` là true hay false.
> > 4. Bubbling: trình duyệt đi lên
> >     - Chạy lên theo thứ tự bé → lớn
> >     - Ở mỗi node, kiểm tra xem có Listener với `capture: false` hoặc mặc định không truyền mới được chạy.
> > [!note]+ ### Làm sao JS theo dõi sự kiện?
> > - Bằng cách dùng phương thức `addEventListener()` để yêu cầu một phần tử trên trang phải theo dõi một sự kiện cụ thể nào đó.
> > - Cú pháp
> > ```javascript
> > element.addEventListener(eventName, eventHandler);
> > ```
> > - VD: Khi người dùng click vào nút, một pop up sẽ hiện lên.
> > ```javascript
> > <button id="btn">Click me!</button>
> > 
> > <script>
> > 	const button = document.getElementById("btn");
> > 	button.addEventListener("click", function () {
> > 	alert("Button clicked!");
> > 	});
> > </script>
> > ```
> > 
> > 
> > [!note]+ ### Cách viết EventHandler - Hàm xử lý
> > - Cách 1: Dùng hàm ẩn danh
> > ```javascript
> > button.addEventListener("click", function () {
> > 	alert("Button clicked!");
> > });
> > ```
> > - Cách 2: Dùng hàm định nghĩa bên ngoài
> > ```javascript
> > function handleClick() {
> > 	alert("Button clicked!");
> > }
> > button.addEventListener("click", handleClick);
> > ```
> > [!note]+ ### Tham số Event
> > > Event thực chất là một đối tượng, khi một sự kiện xảy ra, Javascript sẽ tự động tạo ra một đối tượng bao gồm các chi tiết về sự kiện đó. Đây gọi là **Đối Tượng Sự Kiện**.
> > 
> > Như vậy, ta có thể toàn quyền truy cập tới các thuộc tính như sau:
> > 
> > | Thuộc tính | Ý nghĩa |
> > | --- | --- |
> > | `target` | Element nơi sự kiện xảy ra |
> > | `currentTarget` | Element đang xử lý sự kiện |
> > | `type` | Loại sự kiện |
> > | `bubbles` | Có nổi lên không |
> > | `cancelable` | Có thể chặn không |
> > | `defaultPrevented` | Đã chặn chưa |
> > | `eventPhase` | Đang ở giai đoạn nào |
> > | `timeStamp` | Thời gian xảy ra |
> > | `isTrusted` | Người dùng hay script gây ra |
> > 
> > Ta truyền các Event Object này vào Listner, khi sự kiện được bắt lại, JS sẽ tự động truyền Event Object vào, dựa vào `type` của Event đó.
> > ```javascript
> > element.addEventListener("click", function (event) {
> >     console.log(event); // <-- Browser tự truyền vào
> > });
> > ```
> > Khi ta click:
> > - Trình duyệt tạo một object kiểu `MouseEvent` với `type` = **click**
> > - Sau đó **tự truyền** nó vào **tham số đầu tiên** của function
> > - Có thể ngay lập tức dùng event đó.
> > 
> > **VD thực tế**: truy cập thuộc tính `target` của sự kiện để biến đổi phần tử được nhắm tới
> > ```javascript
> > <button id="myButton">Click me!</button>
> > <script>
> > 	const button = document.getElementById("myButton");
> > 	button.addEventListener("click", function (event) {
> > 		console.log(event); // Full event object
> > 		console.log(event.target); // The element that was clicked
> > 		event.target.style.backgroundColor = "yellow"; // Change background color
> > });
> > </script>
> > ```
> > VD thực tế hơn: Xử lý Form
> > ```javascript
> > <form>
> > 	<input type="text" placeholder="Enter your name" id="nameInput" />
> > 	<input type="email" placeholder="Enter your email" id="emailInput" />
> > </form>
> > 
> > <script>
> > 	const inputs = document.querySelectorAll("input");
> > 	inputs.forEach(function (input) {
> > 		input.addEventListener("focus", function (event) {
> > 			event.target.style.border = "2px solid blue"; // Highlight the input
> > 		});
> > 		input.addEventListener("blur", function (event) {
> > 			event.target.style.border = "1px solid gray"; // Reset when unfocused
> > 		});
> > 	});
> > </script>
> > ```
> > [!note]+ ### Các ví dụ khác
> > - Submit Form
> > ```javascript
> > <form id="loginForm">
> > 	<input type="text" placeholder="Username" required />
> > 	<button type="submit">Login</button>
> > </form>
> > <script>
> > 	const form = document.getElementById("loginForm");
> > 	form.addEventListener("submit", function (event) {
> > 		event.preventDefault(); // Prevent page reload
> > 		alert("Form submitted!");
> > 	// You can now process the form data here
> > 	});
> > </script>
> > ```
> > Tại sao `preventDefault()` lại quan trọng?
> >     - Bình thường khi một form được submit, trình duyệt sẽ tải lại trang (hành vi mặc định).
> >     - Các code Javascript được sinh ra trong quá trình chạy sẽ MẤT khi tải lại trang.
> > Vì vậy, ta dùng `preventDefault()` để:
> >     - Giữ nguyên trang hiện tại.
> >     - Xử lý form điền thủ công (chuẩn hóa, gửi dữ liệu qua Fetch API,…)
> > - Xóa EventListener
> >     - Xóa EventListener như thế nào?
> >         - Ta gọi method `removeEventListener()` để xóa EL đang được gắn vào một phần tử. EL đó sẽ không được kích hoạt nữa
> >         - Cú pháp:
> > ```javascript
> > element.removeEventListener(event, function, options);
> > ```
> >     - Tại sao phải xóa EventListner?
> >         - Trong các loại ứng dụng trên một trang duy nhất (SPA), các phần tử này thường sẽ được lưu lại trên cùng trang, điều này khiến các sự kiện trùng lặp xảy ra chồng nhau trên 2 phần tử khác nhau, các API được gọi đồng thời, lỗi,…
> >         - Khi đó, ta dùng tham số `option` để cài đặt lại EventListener
> > ```javascript
> > button.addEventListener("click", () => {
> > 	alert("Clicked once!");
> > }, { once: true }); //EventListner sẽ bị xóa ngay sau lần chạy đầu
> > ```
> >         - Ta có các option sau:
> > | Option | Kiểu dữ liệu | Ý nghĩa | Default |
> > | --- | --- | --- | --- |
> > | **capture** | boolean | Lắng nghe sự kiện ở **capturing phase** thay vì **bubbling** | `false` |
> > | **once** | boolean | Listener chạy **một lần**, sau đó tự remove | `false` |
> > | **passive** | boolean | Listener **không dùng được **`**preventDefault()**` (tăng hiệu năng) | `false` |
> > | **signal** | AbortSignal | Dùng để **hủy listener** bằng `AbortController` | *không có* |
> > [!note]+ ### Hàm gọi lại (Callback Function)
> > > [!note]+ Định nghĩa
> > > > Một Callback là một hàm được đưa vào như một đối số của một hàm khác. Nói dễ hiểu là một hàm được gọi lại bởi một hàm khác.
> > > 
> > > Đây là cơ sở cho các thao tác bất đồng bộ (asynchronous operations) như:
> > > - Gọi API
> > > - Các sự kiện Click
> > > - Dữ liệu được tải sau một thời gian
> > 
> > > [!note]+ Cú pháp
> > > ```javascript
> > > // Define a callback function
> > > function greeting(name) {
> > > 	console.log('Hello ' + name);
> > > }
> > > // Function that accepts a callback
> > > function processUserInput(callback) {
> > > 	const name = 'Alice';
> > > 	callback(name);
> > > }
> > > // Call function with callback
> > > processUserInput(greeting);
> > > ```
> > 
> > > [!note]+ Thực thi trễ (Delay)
> > > ```javascript
> > > document.getElementById('myButton').addEventListener('click', function () {
> > > 	alert('Button clicked!');
> > > 	setTimeout(function () {
> > > 	console.log('This runs after 3 seconds');
> > > }, 3000);
> > > ```
> > 
> > > [!note]+ Gọi API Bất đồng bộ dữ liệu
> > > ```javascript
> > > function fetchData(callback) {
> > > 	setTimeout(() => {
> > > 		console.log('Data fetched');
> > > 		callback();
> > > 	}, 1000);
> > > }
> > > fetchData(function () {
> > > 	console.log('Now process the data');
> > > });
> > > ```

> > > 
> > > Ta làm việc này khi:
> > > - Áp đặt luật nghiêm ngặt hơn cho dữ liệu
> > > - Kiểm tra hai trường nhập phải giống nhau
> > > - Chuẩn hóa số điện thoại,…
> > > - Xử lý các trường nhập có điều kiện

> > > 1. Theo dõi sự kiện submit của form
> > > 2. Dùng Javascript để lấy ra dữ liệu nhập
> > > 3. Viết logic xử lý dữ liệu
> > > 4. Nếu dữ liệu không hợp lệ:

> > > Kiểm tra mật khẩu
> > > ```javascript
> > > <form id="registerForm">
> > > 	<input type="password" id="password" placeholder="Password" required />
> > > 	<input type="password" id="confirmPassword" placeholder="Confirm Password" required />
> > > 	<button type="submit">Register</button>
> > > </form>
> > > 
> > > <script>
> > > 	const form = document.getElementById("registerForm");
> > > 	form.addEventListener("submit", function (event) {
> > > 		const password = document.getElementById("password").value;
> > > 		const confirmPassword = document.getElementById("confirmPassword").value;
> > > 		if (password !== confirmPassword) {
> > > 			event.preventDefault(); // Stop form submission
> > > 			alert("Passwords do not match!");
> > > 		}
> > > 	});
> > > </script>
> > > ```
> > > Kiểm tra mật khẩu thực tế
> > > ```javascript
> > > form.addEventListener("submit", function (event) {

> > > 	if (!passwordRegex.test(password)) {
> > > 		event.preventDefault();

> > > 		letter, a number, and a special character.");
> > > 	}
> > > });
> > > ```
> [!note]+ # Xử lý lỗi
> > [!note]+ ### Tại sao phải xử lý lỗi
> > Lỗi xảy ra khắp mọi nơi, từ dữ liệu nhập vào, gọi API, đọc File, lỗi đánh máy,…
> > Nếu không xử lý:
> > - Ứng dụng sẽ bị thoát đột ngột
> > - Người dùng không thể biết nguyên nhân lỗi
> > - Độ chuẩn dữ liệu và trải nghiệm người dùng đi xuống
> > 
> > Xử lý lỗi tốt giúp:
> > - Làm cho ứng dụng hoạt động ổn định
> > - Đưa ra thông báo lỗi thân thiện người dùng
> > - Giúp lập trình viên xử lý lỗi dễ dàng
> > [!note]+ ### Cấu trúc cơ bản
> > ```javascript
> > try {
> > 	let result = 10 / x; // x is undefined -> error
> > } catch (error) {
> > 	console.log("An error occurred:", error.message);
> > }
> > ```
> > Ví dụ thực tiễn khi gọi API
> > ```javascript
> > async function fetchProducts() {
> > 	try {
> > 		const response = await fetch('https://api.example.com/products');
> > 		if (!response.ok) {
> > 			throw new Error('Failed to fetch products. Status: ' + response.status);
> > 		}
> > 		const products = await response.json();
> > 		renderProducts(products);
> > 	} catch (error) {
> > 		console.error('Error fetching products:', error.message);
> > 		showErrorMessageToUser("Unable to load products. Please try again later.");
> > 	}
> > }
> > ```
> > Dùng finally
> > ```javascript
> > async function loadData() {
> > 	showLoadingSpinner();
> > 	try {
> > 		const data = await fetchData();
> > 		renderData(data);
> > 	} catch (error) {
> > 		console.error(error);
> > 	} finally {
> > 		hideLoadingSpinner();
> > 	}
> > }
> > ```
> > > [!note] 🗣
> > > Không nên lạm dụng try/catch, chỉ sử dụng ở những nơi code bất ổn định
> > [!note]+ ### Một số Best Practice
> > 1. Phạm vi try/catch phải rõ ràng: chỉ bọc đoạn code tiềm ẩn nguy cơ.
> > 2. Log lỗi một cách rõ ràng.
> > 3. Tận dụng finally: để dọn dẹp thao tác như việc ẩn thanh loading
> > 4. Phân loại lỗi: Xử lý các loại lỗi khác nhau theo các cách khác nhau nếu cần
> [!note]+ # Javascript và các thao tác bất đồng bộ
> > [!note]+ ### Tại sao lại cần xử lý bất đồng bộ?
> > > Dù JS là một ngôn ngữ đơn luồng, vậy nhưng chúng ta thường gặp phải các công việc tốn nhiều thời gian xử lý như:
> > 
> > - Gọi API
> > - Đọc, viết File
> > - Lấy dữ liệu từ DB
> > - Tải ảnh hoặc tài nguyên
> > 
> > Nếu như các công việc này chạy một cách lần lượt (đồng bộ với nhau), trang web sẽ bị đơ vì phải chờ các công việc xử lý. Do đó, Javascript giải quyết vấn đề này với cơ chế bất đồng bộ.
> >  
> > [!note]+ ### Làm thế nào để JS xử lý các thao tác bất đồng bộ?
> > JS sử dụng **Event Loop** để xử lý các công việc bất đồng bộ. Nói dễ hiểu thì:
> > 1. Nó đặt các thao tác này vào hàng chờ
> > 2. Tiếp tục xử lý các đoạn code phía sau
> > 3. Khi một công việc chạy xong, nó sẽ gọi đến công việc tiếp theo

> [!note]+ # Fetch API
> > [!note]+ ### Định nghĩa
> > > Fetch API là một giao diện tân tiến để tạo ra các HTTP request trong JS. Nó thay cho XMLHTTPRequest (XHR) đã cũ với cú pháp dễ sử dụng hơn, dựa vào **Promise**
> > [!note]+ ### Cú pháp cơ bản
> > ```javascript
> > fetch(url, options)
> > 	.then(response => {
> > 		// Process response
> > 		return response.json(); // Or response.text(), response.blob(), etc.
> > 	})
> > 	.then(data => {
> > 		// Process parsed data
> > 	})
> > 	.catch(error => {
> > 		// Handle errors
> > });
> > ```
> > [!note]+ ### GET Request
> > - GET là phương thức HTTP để lấy ra dữ liệu từ máy chủ. Nó là phương thức mặc định của Fetch API
> > - Đặc điểm của GET:
> >     - Được dùng để lấy ra dữ liệu, không thay đổi lên server
> >     - Các tham số được truyền qua URL
> >     - Độ dài URL bị giới hạn (khoảng 2048 kí tự)
> >     - Dữ liệu được gửi ngay trong URL và có thể được lưu vào cache trình duyệt
> > 
> > ```javascript
> > // Basic GET request
> > fetch('https://api.example.com/users')
> > 	.then(response => {
> > 		if (!response.ok) {
> > 			throw new Error(`HTTP error: ${response.status}`);
> > 		}
> > 		return response.json();
> > 	})
> > 	.then(data => {
> > 		console.log('Retrieved data:', data);
> > 	})
> > 	.catch(error => {
> > 		console.error('Error fetching data:', error);
> > });
> > ```
> > [!note]+ ### POST Request
> > - POST là phương thức để gửi dữ liệu đến máy chủ nhằm tạo và cập nhật dữ liệu
> > - Đặc điểm của POST:
> >     - Dùng để gửi dữ liệu đến máy chủ
> >     - Dữ liệu nằm trong `body` của request
> >     - Dữ liệu gửi đi không có giới hạn về kích thước
> >     - Dữ liệu không thể thấy qua URL và không thể lưu cache
> > 
> > ```javascript
> > // POST request with JSON data
> > fetch('https://api.example.com/users', {
> > 	method: 'POST',
> > 	headers: {
> > 		'Content-Type': 'application/json',
> > 		'Authorization': 'Bearer YOUR_TOKEN'
> > 	},
> > 	body: JSON.stringify({
> > 		name: 'John Doe',
> > 		email: 'johndoe@example.com',
> > 		age: 30
> > 	})
> > })
> > .then(response => response.json())
> > .then(data => console.log('User creation result:', data))
> > .catch(error => console.error('Error:', error));
> > ```
