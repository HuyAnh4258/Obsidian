> [!note]+ # SQL Overview
> > [!note]+ ### What is SQL?
> > > [!note] 🗣
> > > SQL stands for Structured Query Language
> > 
> > - It is a computer language to:
> >     - **store **data
> >     - **manipulate **data
> >     - **retrieve **data
> > [!note]+ ### SQL Process
> > ![[image.png]]
> > [!note]+ ### SQL Commands
> > > SQL consists of 3 components:
> > - Data Definition Language (CREATE, ALTER, DROP)
> > - Data Manipulation Language (INSERT, DELETE, UPDATE, SELECT)
> > - Data Control Language (GRANT, REVOKE)
> > 
> >

> [!note]+ # SQL Server Table
> > [!note]+ ### Schema Object
> > - A ‘namespace’ can have objects inside
> > - To further control and manage the objects inside of a namespace, you can put them in sub-groups called schemas.
> > - Schema default:
> >     - dbo is default schema in every database
> >     - Ex: SalesOrderDetail, HumanResources.Department
> >     - [linked-server].[DBName].[SchemaName].[Objectname]
> > - Schema as:
> >     - naming boundaries
> >     - security boundaries
> > [!note]+ ### SQL Server Table (Create, Alter, Drop)
> > > [!note]+ IDENTITY
> > > - Identity have a SEED and an INCREMENT
> > > 
> > > ```sql
> > > --Syntax:
> > > column1 INT IDENTITY(SEED,INCREMENT)
> > > --MSV starts at 1 and increment by 1 for each record
> > > MSV INT IDENTITY(1,1) 
> > > ```
> > 
> > > [!note]+ CREATE TABLE
> > > 1. T-SQL: Create by defining all attributes
> > > 
> > > ```sql
> > > CREATE TABLE tableName (
> > > column_1 datatype [ NULL | NOT NULL ],
> > > column_2 datatype [ NULL | NOT NULL ], ...
> > > );
> > > ```
> > > 2. T-SQL: Create by copying from existing tables
> > > 
> > > ```sql
> > > SELECT (Column 1, …) INTO <New Table name>
> > > FROM <Old Table name>;
> > > ```
> > > 3. Using table designer
> > 
> > > [!note]+ ALTER TABLE
> > > 1. T-SQL: Alter a table by adding new columns
> > > 
> > > ```sql
> > > ALTER TABLE <Table name>
> > > ADD Column1 datatype, Column2 datatype
> > > ```
> > > 2. Use designer
> > 
> > > [!note]+ DROP TABLE
> > > 1. T-SQL: Delete Table
> > > 
> > > ```sql
> > > DROP TABLE <tableName>;
> > > ```
> > > 2. Using SSMS
> > [!note]+ ### SQL Server: Keys & Constraint
> > > [!note]+ Keys
> > > 1. Primary Key
> > >     1. Create a table along with PK
> > > ```sql
> > > CREATE TABLE <Table_Name> (
> > > Column1 datatype,
> > > Column2 datatype,
> > > CONSTRAINT <Name> PRIMARY KEY (Column name) .
> > > );
> > > ```
> > > b.   Add with ALTER TABLE
> > > ```sql
> > > ALTER TABLE tableName
> > > ADD CONSTRAINT constraintName
> > > PRIMARY KEY (column_1, column_2, ... column_n);
> > > ```
> > > 2. Foreign Key
> > >     2. T-SQL: CREATE
> > > ```sql
> > > CREATE TABLE childTable (
> > > column_1 datatype [ NULL |NOT NULL ],
> > > column_2 datatype [ NULL |NOT NULL ], ...
> > > CONSTRAINT fkey_name
> > > FOREIGN KEY (child_column1, ... child_column_n)
> > > REFERENCES parentTable (parent_column1, parent_column2,
> > > ... parent_column_n)
> > > [ON DELETE { NO ACTION |CASCADE |SET NULL |SET DEFAULT }]
> > > [ON UPDATE { NO ACTION |CASCADE |SET NULL |SET DEFAULT }]
> > > );
> > > ```
> > > b.   T-SQL: ALTER
> > > ```sql
> > > ALTER TABLE ChildTable
> > > ADD CONSTRAINT fkey_name FOREIGN KEY (child_column1, child_column2, ... child_column_n)
> > > REFERENCES parentTable (parent_column1, parent_column2, ... parent_column_n);
> > > ```
> > > c.   SSMS
> > > 3. Composite Key
> > > > A composite key is made by the combination of two or more columns in a table that can be used to uniquely identify each row in the table when the columns are combined uniqueness of a row is guaranteed.
> > 
> > > [!note]+ Constraint
> > > 1. UNIQUE
> > > ```sql
> > > --Defining constraint in CREATE TABLE
> > > CREATE TABLE table_name (
> > > column1 datatype [ NULL | NOT NULL ],
> > > column2 datatype [ NULL | NOT NULL ], ...
> > > CONSTRAINT constraint_name UNIQUE (uc_col1, ... uc_col_n)
> > > );
> > > 
> > > --ALTER TABLE
> > > ALTER TABLE table_name
> > > ADD CONSTRAINT constraint_name
> > > UNIQUE (column1, column2, ... column_n);
> > > 
> > > --DROP
> > > ALTER TABLE table_name DROP CONSTRAINT constraint_name
> > > ```
> > > 2. CHECK
> > > ```sql
> > > --Defining constraint in CREATE TABLE
> > > CREATE TABLE table_name (
> > > column1 datatype [ NULL | NOT NULL ],
> > > column2 datatype [ NULL | NOT NULL ], ...
> > > CONSTRAINT constraint_name
> > > CHECK [ NOT FOR REPLICATION ] (column_name condition)
> > > );
> > > 
> > > --ALTER TABLE
> > > ALTER TABLE table_name
> > > ADD CONSTRAINT constraint_name 
> > > CHECK (column_name condition);
> > > 
> > > --DROP
> > > ALTER TABLE table_name DROP CONSTRAINT constraint_name;
> > > ```
> > [!note]+ ### TRUNCATE Statement
> > - Remove all rows (records) in a table
> > - Table structures remain
> >     - Also resets the identity value
> >     - Release memory
> > - DROP vs. TRUNCATE:
> >     - TRUNCATE is faster and ideal to delete all data from table
> >     - TRUNCATE preserves table structure, DROP does not
> >     - DROP cannot be rolled back
