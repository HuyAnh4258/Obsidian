> [!note]+ # SQL JOINS
> > [!note]+ ### What’s SQL JOINS?
> > - SQL Joins are used to combine rows from two or more tables based on logical relationships between the tables.
> > - Types of JOIN in SQL:
> >     - Inner Join
> >     - Outer Join
> >     - Cross Join
> >     - Self Join
> > [!note]+ ### INNER JOIN
> > - The INNER JOIN selects all rows from BOTH tables as long as there is a
> > match between the columns in both tables.
> > - Eliminate the rows that do not match with a row from the other table
> > 
> > ```sql
> > SELECT col_names
> > FROM Table_A A INNER JOIN Table_B B
> > ON A.Col1 = B.Col1
> > ```
> > [!note]+ ### LEFT OUTER JOIN
> > - Return all of the records in the left table (table A) regardless if any of
> > those records has a match in the right table (table B)
> > 
> > ```sql
> > SELECT col_names
> > FROM Table_A A LEFT JOIN Table_B B
> > ON A.Col1 = B.Col1
> > ```
> > [!note]+ ### RIGHT OUTER JOIN
> > - Return all of the records in the right table (table B) regardless if any of
> > those records have a match in the left table (table A)
> > 
> > ```sql
> > SELECT col_names
> > FROM Table_A A RIGHT JOIN Table_B B
> > ON A.Col1 = B.Col1
> > ```
> > [!note]+ ### FULL OUTER JOIN
> > - Return all of the records from both tables, joining records from the left
> > table (table A) that match records from the right table (table B)
> > 
> > ```sql
> > SELECT col_names
> > FROM Table_A A FULL JOIN Table_B B
> > ON A.Col1 = B.Col1
> > ```
> > [!note]+ ### CROSS JOIN
> > - Return records that are multiplication of record number from both the
> > tables
> > 
> > ```sql
> > SELECT col_names
> > FROM Table_A A CROSS JOIN Table_B B
> > ```
> > [!note]+ ### SELF JOIN
> > - A SELF JOIN is a join of a table to itself. In SELF JOIN, we can use:
> >     - INNER JOIN
> >     - OUTER JOIN
> >     - CROSS JOIN
> > [!note]+ ### (LEFT, RIGHT, OUTER) EXCLUDING JOIN
> > - Return all unmatched records with selected excluded area.
> > [!note]+ ### Joining 3+ or more tables
> > - Since FROM clauses can contain multiple join specifications, this allows
> > many tables to be joined for a single query.
> > 
> > ```sql
> > SELECT col_names
> > FROM Table_A A JOIN Table_B B
> > ON A.Col1 = B.Col1 LEFT JOIN Table_C C
> > ON B.Col2 = C.Col2
> > ...
> > ```
> [!note]+ # Sub-Queries
> > [!note]+ ### What is a subquery?
> > > A sub-query, also called an inner query, is a SQL query nested inside a
> > larger query.
> > SQL Server support nesting up to 32 levels, others up to 255 or 1000+ (SQLite)
> > 
> > - The subquery can be nested inside a SELECT, INSERT, UPDATE, or
> > DELETE statement or inside another subquery.
> > - You can use the comparison operators, such as >, <, or =. The comparison operator can also be a multiple-row operator, such as IN, ANY, or ALL.
> > 
> > ```sql
> > SELECT select_list
> > FROM table
> > WHERE expr operator (SELECT select_list FROM table)
> > ```
> > ```sql
> > SELECT SUM (Sales) AS Sale_Sum FROM Store_Information
> > WHERE Store_Name IN
> > (SELECT Store_Name FROM Geography
> > WHERE Region_Name = 'West');
> > ```
> > [!note]+ ### Types of Subquery
> > - Single row
> > ```sql
> > SELECT agent_name, agent_code, phone_no
> > FROM agents
> > WHERE agent_code = 
> > (SELECT agent_code FROM agents WHERE agent_name = 'Alex')
> > ```
> > - Multiple row
> > ```sql
> > SELECT ord_num, ord_amount, ord_date, cust_code, agent_code
> > FROM orders
> > WHERE agent_code IN
> > (SELECT agent_code FROM agents WHERE working_area='Sydney')
> > ```
> > - Multiple column
> > ```sql
> > SELECT ord_num, agent_code, ord_date, ord_amount
> > FROM orders o1
> > WHERE EXISTS(
> > SELECT agent_code, ord_amount
> > FROM orders o2
> > WHERE o1.agent_code = o2.agent_code
> > 	AND o1.ord_amount = o2.ord_amount)
> > ORDER BY ord_amount ASC
> > ```
> > - Correlated
> > ```sql
> > SELECT * FROM orders o
> > WHERE agent_code IN (
> > SELECT agent_code FROM agents a
> > WHERE o.ship_city = a.working_area)
> > ```
> > - Nested
> > ```sql
> > SELECT *
> > FROM orders
> > WHERE ship_city IN (SELECT DISTINCT working_area 
> > FROM agents WHERE agent_code
> > IN
> > (SELECT agent_code FROM agents WHERE commission >= 0.14))
> > ```
> > [!note]+ ### Rules:
> > - You must enclose a subquery in parenthesis.
> > - A subquery must include a SELECT clause and a FROM clause.
> > - Subqueries that return more than one row can only be used with multiple value operators, such as the IN operator.
> > - A subquery can include optional WHERE, GROUP BY, and HAVING
> > clauses.
> > - A subquery cannot include COMPUTE or FOR BROWSE clauses.
> > - You can include an ORDER BY clause only when a TOP clause is
> > included.
> > - You can nest subqueries up to 32 levels.
> [!note]+ # CTE and Ranking Functions
> > [!note]+ ### Common Table Expressions
> > - A CTE can be thought of as a temporary result set that is defined within the execution scope of a single SELECT, INSERT, UPDATE, DELETE.
> > 
> > ```sql
> > WITH CTE_Name [ col_names]
> > AS
> > (
> > CTE_query_definition
> > )
> > ```
> > ```sql
> > ; WITH cte_Agents(agent_name, working_area, amount_of_agent)
> > (AS
> > SELECT a.agent_name, a.working_area, COUNT(o.agent_code) 
> > AS AMOUNT_AGENT
> > FROM dbo.agents a INNER JOIN dbo.orders o
> > ON A.agent_code = O.agent_code
> > )GROUP BY a.agent_name, a.working_area
> > SELECT * FROM cte_Agents cte
> > WHERE cte.working_area = 'Bungnlme'
> > ```
> > [!note]+ ### Ranking Functions
> > > [!note]+ ### What are Ranking Functions?
> > > - Ranking functions  provide the ability to rank each row of data
> > > - There are 4 kinds of ranking functions:
> > >     - ROW_NUMBER
> > >     - RANK
> > >     - DENSE_RANK
> > >     - NTILE
> > > [!note]+ ### ROW_NUMBER
> > > - Returns the sequential number of a row within a partition of
> > > a result set
> > > 
> > > ```sql
> > > SELECT FullName, Subject, Marks, ROW_NUMBER() 
> > > OVER(ORDER BY FullName) RowNumber
> > > FROM ExamResult
> > > ORDER BY FullName, Subject
> > > ```
> > > [!note]+ ### RANK
> > > - Returns the rank of each row within the partition of a result set
> > > 
> > > ```sql
> > > SELECT FullName, Subject, Marks, RANK() OVER(PARTITION BY
> > > FullName ORDER BY Marks DESC) Rank
> > > FROM ExamResult
> > > ORDER BY FullName, Subject
> > > ```
> > > [!note]+ ### DENSE_RANK
> > > - Returns the rank of rows within the partition of a result set, without
> > > any gaps in the ranking
> > > 
> > > ```sql
> > > SELECT
> > > FullName, Subject, Marks, DENSE_RANK()
> > > OVER (PARTITION BY FullName ORDER BY Marks DESC) Rank
> > > FROM ExamResult
> > > ORDER BY FullName
> > > ```
> > > [!note]+ ### NTILE
> > > - Distributes the rows in an ordered partition into a specified number of groups
> > > 
> > > ```sql
> > > SELECT FullName, Subject, Marks,
> > > NTILE(2) OVER
> > > (ORDER BY Marks DESC)Quartile
> > > FROM ExamResult
> > > ```
