> [!note]+ # OOPs Concepts
> > [!note]+ ### What is a Class?
> > > A `class` is the <u>blueprint</u> to create `objects`.
> > 
> > - For example:
> > 
> > ```java
> > public class House {
> > 	String address; //These are attributes for the object
> > 	String color;
> > 	double are;
> > 	void openDoor() {
> > 	// TODO: This is a Class's method
> > 	}
> > 	void closeDoor() {
> > 	// TODO: Class method is the behaviour of a class
> > 	}
> > }
> > ```
> > - Creating `objects` from `class`:
> > 
> > ```java
> > public class HouseManagement {
> > 	public static void main(String[] args) {
> > 		House house1 = new House("Duytan", "Blue", 1000);
> > 		House house2 = new House("Tonthatthuyet", "Green", 1200);
> > 		System.out.println(house1.address + "\t" + house1.color +
> > 		"\t" + house1.are);
> > 		System.out.println(house2.address + "\t" + house2.color +
> > 		"\t" + house2.are);
> > 	}
> > }
> > ```
> > 
> > - Another example:
> > 
> > ![[image 10.png]]
> > 
> > - Syntax: Declaring a Class
> > ```java
> > [public][<abstract><final>]class <ClassName>
> > [extends <SuperClass>] [implements <InterfaceName>]{
> > <Attribute/Field declarations { initialization code }>
> > <Constructors>
> > <Methods>
> > }
> > ```
> > [!note]+ ### Constructors
> > > A constructor is the called to create an object using the `new` operator
> > 
> > ```java
> > public class Computer {
> >     String brand;
> >     String name;
> >     int ram;
> >     int storage;
> >     String color;
> > 
> >     //Constructor - Java will default create one (with no param) if there is none
> >     public Computer(){
> >         System.out.println("Creating a new Computer with normal Constructor");
> >     }
> >     
> >     //Parametered Constructor
> >     public Computer(String brand, String name, int ram, int storage, String color){
> >         //'this' refer to the object we create
> >         System.out.println("Creating a new Computer with param Constructor");
> >         this.brand = brand;
> >         this.name = name;
> >         this.ram = ram;
> >         this.storage = storage;
> >         this.color = color;
> >     }
> > 
> >     //Methods
> >     public void bootUp(){
> >         System.out.println("Booting up computer");
> >     }
> > 
> >     public void shutdown(){
> >         System.out.println("Shutting down computer");
> >     }
> > 
> >     public void playGame(){
> >         System.out.println("Playing game with computer");
> >     }
> > }
> > ```
> > - Constructor is a block of code that initializes the newly created object.
> >     - `Constructor` has same name as the `Class`
> >     - People often refer constructor as `special type of method` in Java. It doesn’t have a return type (even `void`)
> >     - You can create multiple constructors, with diff. param. (`Constructor Overloading`)
> >     - If you don't write any constructor, the compiler (Java) will write one for you.
> >     - If you include any constructors in a class, the compiler will not create a default
> > constructor!
> >     - Nếu không viết, Java sẽ tự tạo constructor mặc định không tham số.
> >     - Nếu tự viết ít nhất một constructor, Java sẽ không tự tạo constructor mặc định nữa.
> > 
> > 
> > 
> > > [!note] 🗣
> > > Java **DOES NOT** have a `Destructor` (like C++), `JVM Garbage Collector` takes care of destroying objects automatically.
> > [!note]+ ### Instance Variable (Field)
> > > Instance variable in Java is used by objects to store their states and info.
> > Fields (data members) can be any primitive or reference type
> > 
> > - Syntax:
> > 
> > ```java
> > [Access modifier] <Data type> <field_name>;
> > ```
> > - Access modifiers:
> > 
> > | Modifier | Class | Package | Subclass (Child-class) | Outside Package |
> > | --- | --- | --- | --- | --- |
> > | public | Y | Y | Y | Y |
> > | protected | Y | Y | Y | N |
> > | default | Y | Y | N | N |
> > | private | Y | N | N | N |
> > [!note]+ ### Instance method (Behaviour)
> > > Instance method are methods that are tied to innitialized objects.
> > Before calling an instance method, you must create the object.
> > 
> > - Access modifiers (same as instance variable):
> > 
> > | Modifier | Class | Package | Subclass (Child-class) | Outside Package |
> > | --- | --- | --- | --- | --- |
> > | public | Y | Y | Y | Y |
> > | protected | Y | Y | Y | N |
> > | default | Y | Y | N | N |
> > | private | Y | N | N | N |
> [!note]+ # `static` Keyword in Java
> > [!note]+ ### Static variables
> > | **Static variables** | **Instance variables** |
> > | --- | --- |
> > | `static variables` are called `class variables` |   |
> > | A static field exist among multiple objects of the class, no matter how they are created | Tied to a specific object, not shared among objects |
> > 
> > - Example:
> > 
> > ```java
> > class Student {
> >         int rollno; //instance variables, one object have one of these
> >         String name;
> >         static String college; //this will be shared among objects of the class
> >         //static block: always run first and only once
> >         static {
> >             college = "ITS"; //setting college for the class
> >             //every student created will have this information
> >             System.out.println("Static block");
> >         }
> > 
> >         Student(int rollno, String name) {
> >             this.rollno = rollno;
> >             this.name = name;
> >             System.out.println("Constructor block");
> >         }
> > 
> >         void display() {
> >             System.out.println(rollno + " " + name + " " + college);
> >         }
> > 
> > 				//All students now share this method
> >         static void changeCollege() {
> >             college = "FU";//change all student's college
> >         }
> >     }
> > ```
> > ```java
> > public static void main(String args[]) {
> > 	// Student.changeCollege();
> > 	Student s1 = new Student(111, "Karan");
> > 	Student s2 = new Student(222, "Aryan");
> > 	Student.changeCollege();
> > 	s1.display();
> > 	s2.display();
> > }
> > ```
> > - Comparison:
> > 
> > | Static Variables | Non-Static Variables |
> > | --- | --- |
> > | They can access them using `class names`. | They can be accessed only `using objects` |
> > | They can access them with `static methods`, along with `non-static methods` | They can be accessed `only using non-static methods` |
> > | Tehy are allocated to memory `only once` while innitiating the class | A `memory per object` is allocated |
> > | These variables are `shared by all the objects` or instances of the class | `Each object has its own copy` of the non-static variables |
> > | Static variables have `global scope` | They have `local scope` |
> > [!note]+ ### Static methods
> > > Similar to `static variable`, `a static method` can be used without creating an object and is shared among objects of the same class.
> > 
> > - Syntax:
> > 
> > ```java
> > static [return_type] method_name();
> > ```
> > 
> > | Static Methods | Non-Static Methodsnon-Static Methods |
> > | --- | --- |
> > | These methods support early or `compile-time binding` | They support late, `run-time`, or `dynamic binding` |
> > | These methods can `only access static variables` of other classes as well as their own class. | They can access both static as well as `non-static` members |
> > | You `can’t override static methods` | They `can be overridden` |
> > | `Less memory consumption` since they are allocated memory only once when the class is being loaded. | Memories are `allocated for each object` |

> [!note]+ # Khái niệm về `Heap` và `Stack`
> > Bộ nhớ **Stack** và **Heap** là hai khu vực bộ nhớ quan trọng mà **Máy ảo Java (JVM)** sử dụng để chạy ứng dụng Java một cách tối ưu.
> 
> JVM phân chia bộ nhớ thành Stack và Heap, và khi chúng ta khai báo các biến, đối tượng mới, gọi phương thức mới, khai báo chuỗi (`String`), hoặc thực hiện các hoạt động tương tự, JVM sẽ chỉ định bộ nhớ cho các hoạt động này từ **Stack Memory** hoặc **Heap Space**
> > [!note]+ ### Stack
> > > Stack là nơi Java lưu trữ **thông tin tạm thời liên quan đến lời gọi hàm (method call)**
> > 
> > **Ví dụ:**
> > ```java
> > void test() {
> >     int x = 5;        // x nằm trong Stack
> >     String s = "Hi";  // s là reference nằm trong Stack
> > }
> > ```
> > ### Đặc điểm:
> > - Lưu **biến cục bộ** (local variables)
> > - Lưu **tham chiếu đối tượng** (reference), nhưng **không lưu bản thân đối tượng**
> > - Cấu trúc **Last In – First Out (LIFO)**: hàm gọi sau sẽ được xử lý trước
> > - Tự động **giải phóng** khi method kết thúc
> > - Kích thước nhỏ hơn Heap
> > - Truy cập **rất nhanh**
> > 
> > > [!note] 🗣
> > > Stack có lỗi StackOverflowError, xảy ra khi chương trình đệ quy vô hạn
> > 
> > ```java
> > void go() { go(); } // Gọi hoài không dừng → Stack đầy
> > ```
> > [!note]+ ### Heap
> > > Heap là nơi lưu **đối tượng (objects)** và **dữ liệu động**.
> > 
> > **Ví dụ:**
> > ```java
> > Person p = new Person(); // p ở Stack, đối tượng Person ở Heap
> > ```
> > ### Đặc điểm:
> > - Lưu **đối tượng** được tạo bằng `new`
> > - Lưu **object** mà reference trong Stack trỏ đến
> > - Dung lượng lớn hơn Stack
> > - Truy cập chậm hơn Stack
> > - Vùng nhớ này **được dọn dẹp bởi Garbage Collector (GC)** → tự động thu hồi khi không còn reference
> > 
> > > [!note] 🗣
> > > Heap có lỗi OutOfMemoryError, xảy ra khi tạo quá nhiều object lớn hoặc giữ reference quá lâu
> > 
> > ```java
> > List<int[]> list = new ArrayList<>();
> > while (true) list.add(new int[1_000_000]);
> > ```
