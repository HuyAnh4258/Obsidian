> [!note]+ # Java JDBC Tutorial
> > [!note]+ ### Tổng quan
> > > `JDBC API` hay Java Database Connectivity cho phép Java `kết nối` đến các cơ sở dữ liệu. Việc truy vấn là như nhau giữ các cơ sở dữ liệu khác nhau.
> > 
> > JVM sử dụng JDBC driver để dịch JDBC gọi đến các DB
> > - JDBC sử dụng driver để kết nối đến database
> > - Bốn loại driver của JDBC
> >     - JDBC-ODBC Bridge
> >     - Native Driver
> >     - Network Protocol Driver
> >     - Thin Driver
> > - JDBC driver là các class triển khai lại interface JDBC cho các loại DB khác nhau
> > [!note]+ ### JDBC-ODBC driver cầu nối
> > > Sử dụng `ODBC` driver (Open Database Connectivity), chuyển đổi JDBC thành ODBC
> > 
> > ![[image 25.png]]
> > Không được khuyến khích vì driver mỏng
> > Từ Java 8 thì Oracle không còn hỗ trợ ODBC, khuyến khích JDBC hơn.
> > [!note]+ ### Native-API driver
> > > `Native API` sử dụng các thư viện khác, chuyển thể JDBC thành các lần gọi cục bộ
> > 
> > ![[image 26.png]]
> > Không thuần Java
> > - Ưu điểm: Chạy tốt hơn cầu nối JDBC - ODBC
> > - Nhược điểm: Cần phải cài đặt và có thư viện db trên mọi máy khách
> > [!note]+ ### Network Protocol Driver
> > > `Network Protocol` sử dụng phần mềm trung gian để chuyển JDBC thành một giao thức. Được viết hoàn toàn bằng Java
> > 
> > ![[image 27.png]]
> > - Ưu điểm
> >     - Không cần thư viện khác khi mà ứng dụng web có thể audit, load balance, logging..
> > - Nhược điểm
> >     - Cần có mạng internet trên máy khách
> >     - Cần có code liên quan đến DB ở phần mềm trung gian
> >     - Khó bảo trì hơn vì liên quan để DB ở phần mềm trung gian
> > [!note]+ ### Thin Driver
> > > `Thin driver` biến đổi JDBC trực tiếp thành giao thức, được viết hoàn toàn bằng Java
> > 
> > ![[image 28.png]]
> > - Ưu điểm: Chạy tốt hơn các loại driver khác, không cần phần mềm trên máy khách
> > - Nhược điểm: Driver phụ thuộc vào CSDL

> > 2. Bắt `Connection`
> > 3. Tạo `Statement`
> > 4. Thực thi `Query`
> > 5. Đóng `Connection`
