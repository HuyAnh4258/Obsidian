> [!note]+ # Ngoại lệ trong Java
> > [!note]+ ### Ngoại lệ là gì?
> > > `Ngoại lệ` là một sự kiện không mong muốn và không thể lường trước, xảy ra khi chương trình đang chạy, làm cho luồng thông thường của chương trình không đúng.
> > 
> > - Khi một ngoại lệ xảy ra trong một phương thức, nó sẽ bắn ra một `đối tượng ngoại lệ`
> > - Một ngoại lệ sẽ bao gồm các thông tin về ngoại lệ đó như:
> >     - `Tên` và `miêu tả` của ngoại lệ
> >     - `Trạng thái của chương trình` khi ngoại lệ đó xảy ra
> > [!note]+ ### Lỗi là gì?
> > > `Lỗi` miêu tả một trạng thái không thể phục hồi của chương trình, ví dụ như JVM hết, tràn bộ nhớ, lỗi tương thích thư viện, đệ quy vô tận,…
> > 
> > - Các lỗi thường nằm ngoài khả năng kiểm soát của lập trình viên, vậy nên ta không nên cố gắng xử lý `lỗi`.
> > - Phân biệt giữa `Ngoại lệ` và `Lỗi`
> >     - `Ngoại lệ` là một trạng thái mà một chương trình sẽ gặp phải (do lập trình và lập trình viên có thể kiểm soát được)
> >     - `Lỗi` là một vấn đề nghiệm trọng mà chương trình không nên gặp phải
> > [!note]+ ### Tại sao lại xảy ra ngoại lệ?
> > Các lý do và nguyên nhân thường gặp
> > - Người dùng nhập vào dữ liệu không hợp lệ
> > - Thiết bị bị hỏng hóc
> > - Bị mất kế nối mạng
> > - Giới hạn vật lý (Bộ nhớ)
> > - Lỗi lập trình
> > - Mở một File không khả dụng
> > [!note]+ ### Xử lý ngoại lệ
> > > Việc xử lý ngoại lệ trong Java là một trong những cách hữu hiệu nhất để đối phó với các lỗi runtime, giúp cho chương trình hoạt động đúng như luồng mà nó được đặt ra.
> > 
> > - Đây là cơ chế giúp ta đối phó với các ngoại lệ như:
> >     - ClassNotFoundException
> >     - IOException
> >     - SQLException
> >     - RemoteException
> >     - …
> > 
> > 
> > Ví dụ của ngoại lệ
> > ```java
> > public class Exercise {
> > 	public static void main(String[] args) {
> > 		double side;
> > 		Scanner scnr = new Scanner(System.in);
> > 		System.out.println("Square Processing");
> > 		System.out.print("Enter Side: ");
> > 		side = scnr.nextDouble();
> > 		System.out.println("\nSquare Characteristics");
> > 		System.out.printf("Side: %.2f\n", side);
> > 		System.out.printf("Perimeter: %.2f\n", side * 4);
> > 	}
> > }
> > ```

> [!note]+ # Xử lý ngoại lệ
> > [!note]+ ### Tổng quan
> > > Khi xuất hiện điều kiện để một ngoại lệ sinh ra, Java sẽ ném ra một đối tượng đại diện cho ngoại lệ đó trong phần phương thức mà ngoại lệ đó xảy ra.
> > 
> > - Việc xử lý ngoại lệ xoay quanh 5 keyword sau:
> > > [!note]+ try
> > > > Khối lệnh `try` sẽ bao gồm các câu lệnh mà `có nguy cơ xảy ra ngoại lệ`
> > > 
> > > ```java
> > > try{
> > > 	// statement(s) that might cause exception
> > > }
> > > ```
> > 
> > > [!note]+ catch
> > > > Khối lệnh `catch` sẽ dùng để xử lý ngoại lệ (nếu xảy ra) bên trong khối `try`
> > > 
> > > ```java
> > > catch (Exception_Class_Name ref) {
> > > 	// statement(s) that handle an exception
> > > 	// examples, closing a connection, closing
> > > 	// file, exiting the process after writing
> > > 	// details to a log file.
> > > }
> > > ```
> > 
> > > [!note]+ throw
> > > > `throw` là keyword được dùng để chủ động ném ra một ngoại lệ, thường dùng để thực thi khối `catch` ngay lập tức
> > 
> > > [!note]+ throws
> > > > `throws` là keyword dùng để `khai báo` một ngoại lệ mà lập trình viên cho rằng sẽ xảy ra bên trong một `method`, khi ta gọi đến các method này, việc xử lý các ngoại lệ đã được khai báo là bắt buộc.
> > 
> > > [!note]+ finally
> > > > `finally` là khối lệnh luôn luôn chạy cuối cùng sau khối lệnh `try - catch`
> > > 
> > > Ta thường dùng khối lệnh `finally` để chạy một đoạn mã chung, kể cả khi chương trình không gặp lỗi.
> > > 
> > 
> > Cú pháp hoàn chỉnh của một khối `try - catch - finally`
> > ```java
> > try {
> > //code that may throw an exception
> > } catch(Exception_Class_Name ref){
> > 	//sysout: something happened!
> > } finally {
> > 	//sysout: the exception handling is done
> > }
> > ```
> > ![[image 32.png]]
> > [!note]+ ### Khối `multi-catch`
> > > Một khối `try` có thể có `nhiều khối catch` để bắt nhiều ngoại lệ khác nhau xảy ra
> > 
> > Mỗi một khối catch đều phải bắt một ngoại lệ khác nhau
> > > [!note] 🗣
> > > Ghi nhớ:
> > > - Khi có ngoại lệ xảy ra, chỉ có ngoại lệ đó được bắt và chỉ có một khối catch được thực thi, các đoạn code sau đó đều bị bỏ qua.
> > > - Tất cả khối catch phải được sắp xếp theo thứ tự từ bé đến lớn theo phạm vi bắt ngoại lệ của loại ngoại lệ đó.
> > 
> > ```java
> > public class MultipleCatchBlock {
> > 	public static void main(String[] args) {
> > 		try {
> > 			int a[] = new int[5];
> > 			a[5] = 30 / 0;
> > 			System.out.println(a[10]);
> > 		} catch (ArithmeticException e) {
> > 			System.out.println("Arithmetic Exception occurs"); //caught
> > 		} catch (ArrayIndexOutOfBoundsException e) {
> > 			System.out.println("ArrayIndexOutOfBounds Exception occurs");
> > 		} catch (Exception e) {
> > 			System.out.println("Parent Exception occurs");
> > 		}
> > 		System.out.println("Rest of the code");
> > 	}
> > }
> > ```
> > [!note]+ ### Khối `finally`
> > > Khối `finally` thường được dùng để thực thi các câu lệnh quan trong như việc đóng tài nguyên,… Khối này sẽ luôn được thực thi bất kể có ngoại lệ xảy ra hay là không.
> > 
> > ```java
> > Connection conn= null;
> > try {
> > 	conn= get the db conn;
> > //do some DML/DDL
> > } catch(SQLException ex) {
> > 	//something happened when init conn
> > } finally {
> > 	conn.close();//closing conn
> > }
> > ```
> [!note]+ # Ngoại lệ được kiểm tra và không được kiểm tra
> > [!note]+ ### Ngoại lệ được kiểm tra
> > > `Checked exceptions` là các ngoại lệ được kiểm tra khi chương trình được biên dịch (ở đây là khi Javac thực thi việc biên dịch). 
> > 
> > Khi đó, phương thức có ngoại lệ sẽ bắt buộc phải xử lý ngoại lệ đó hoặc khai báo nó để các phương thức gọi đến nó xử lý.
> > Có 2 loại ngoại lệ được kiểm tra:
> > - Fully checked:
> > Một ngoại lệ được kiểm tra toàn diện là một ngoại lệ mà mọi ngoại lệ con của nó cũng được kiểm tra như `IOException` `InterruptedException`
> > - Partially checked:
> > Một ngoại lệ được bán kiểm tra là khi có một số ngoại lệ con của nó không được kiểm tra đến, ví dụ như `Exception`
> > [!note]+ ### Ngoại lệ không được kiểm tra
> > > `Unchecked Exceptions` hay `Runtime exceptions` là các ngoại lệ mà trình biên dịch không kiểm tra đến, dù cho lập trình viên đã xử lý hay chưa. Đây là trách nhiệm của lập trình viên.
> > 
> > Nếu như các lỗi này không được bắt và xử lý, chương trình sẽ vẫn biên dịch và chạy được mà không ném ra lỗi biên dịch
> > Ví dụ:
> > - ArithmeticException
> > - ArrayIndexOutOfBoundsException
> > - NullPointerException
> > - NegativeArraySizeException
> > 
> > ```java
> > public class ArrayListSample {
> > 	public static void main(String[] args) {
> > 		List<String> list = new ArrayList<>();
> > 		list.add("Apple");
> > 		list.add("Samsung");
> > 		list.add(1, "Oppo");
> > 		// Inserts the specified element at index is 5
> > 		list.add(5, "Nokia");
> > 		// Get size of list
> > 		int size = list.size();
> > 		// Get element in list
> > 		for (int i = 0; i < size; i++) {
> > 			System.out.print(list.get(i) + "\t");
> > 		}
> > 	}
> > }
> > ```

> [!note]+ # Từ khoá `throw` và `throws`
> > [!note]+ ### `throw`
> > > Từ khoá `throw` dùng để ném ra ngay lập tức một ngoại lệ (dù nó đã xảy ra hay chưa)
> > 
> > Chúng ta có thể ném ra cả `checked` và `unchecked` exception.
> > Thường thì từ khoá `throw` chỉ được dùng để ném ra những lỗi tuỳ chỉnh của người dùng
> > 
> > **Cú pháp:**
> > ```java
> > //throw existing exception
> > throw exception;
> > // throw new custom exception
> > throw new IOException("sorry device error");
> > ```
> > 
> > **Ví dụ:**
> > ```java
> > public class ThrowExcep {
> > 	static void fun() {
> > 		try {
> > 			throw new NullPointerException("demo");
> > 		} catch (NullPointerException e) {
> > 			System.out.println("Caught inside fun().");
> > 			// rethrowing the exception
> > 			throw e;
> > 			}
> > 		}
> > 	public static void main(String args[]) {
> > 		try {
> > 			fun();
> > 		} catch (NullPointerException e) {
> > 		System.out.println("Caught in main.");
> > 		}
> > 	}
> > }
> > 
> > //Output:
> > //Caught inside fun().
> > //Caught in main.
> > ```
> > 
> > [!note]+ ### `throws`
> > > `throws` là từ khoá đặt bên cạnh `định danh phương thức` (tên method, tham số truyền vào,…) nhằm chỉ định rằng phương thức đó có thể sẽ ném ra một trong các ngoại lệ được liệt kê với từ khoá `throws`. Những hàm khác khi gọi đến phương thức này sẽ luôn phải xử lý ngoại lệ đã chỉ định sẵn trên phướng thức.
> > 
> > > [!note] 🗣
> > > Thường được dùng để xử lý các ngoại lệ đã được kiểm tra (tránh compiler báo lỗi, không cho chạy code)
> > 
> > **Cú pháp:**
> > ```java
> > return_type method_name() throws Exception_Class_Name {
> > 	//method code
> > }
> > ```
> > [!note]+ ### So sánh giữa `throw` và `throws`
> > |   | throw | throws |
> > | --- | --- | --- |
> > | Công dụng | Dùng để `ném` ra ngoại lệ:<br>Chủ động tạo và quăng ra một ngoại lệ để bắt và xử lý ngay trong lúc chạy | Dùng để `khai báo` ngoại lệ: báo trước rằng method sẽ ném ra ngoại lệ (dù không xử lý nó) |
> > | Checked Exception | Không đủ để ném checked exception ra ngoài method | Cho phép ném checked exception ra khỏi method và xử lý ở caller |
> > | Theo sau bởi | Một đối tượng (instance) đại diện cho ngoại lệ đó | Tên của Class ngoại lệ |
> > | Vị trí sử dụng | Bên trong thân method | Bên cạnh định danh hàm |
> > | Số lượng ngoại lệ cho phép | Chỉ được ném 1 đối tượng ngoại lệ tại một thời điểm khi đang chạy | Được khai báo nhiều loại exception khác nhau |
